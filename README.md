# Poly Panic

Консольный мониторинг рынков Polymarket.

Скрипт:
- опрашивает активные рынки Polymarket;
- сохраняет снимки в SQLite;
- ищет аномалии по объему и цене;
- выводит результат в терминал;
- пишет технические логи в файл.

## Что нужно

- Python 3.10+
- созданный `venv`
- установленные зависимости из `requirements.txt`

## Основные файлы

- `app.py` — точка входа
- `poly_panic/config.py` — все основные настройки
- `data/poly_panic.db` — база со снапшотами рынков
- `logs/poly_panic.log` — лог-файл

## Настройка

Для личного использования удобнее менять значения прямо в `poly_panic/config.py`.

Самые важные:
- `DEFAULT_POLL_INTERVAL_SECONDS` — как часто опрашивать API
- `DEFAULT_MIN_VOLUME_NUM` — минимальный общий объем рынка
- `DEFAULT_WHALE_VOLUME_DELTA` — порог для алерта по росту объема
- `DEFAULT_EXCLUDE_SPORT_MARKETS` — исключать ли спорт
- `DEFAULT_REQUIRED_KEYWORDS` — обязательные ключевые слова
- `DEFAULT_EXCLUDED_KEYWORDS` — слова для исключения рынков
- `DEFAULT_LOG_LEVEL` — уровень логирования

`.env` можно использовать только если хочешь переопределять настройки без правки кода.

## Запуск

Один проход:

```powershell
venv\Scripts\python.exe app.py --once
```

Один проход + показать top:

```powershell
venv\Scripts\python.exe app.py --once --top
```

Постоянный режим:

```powershell
venv\Scripts\python.exe app.py
```

Остановка постоянного режима:

```powershell
Ctrl+C
```

## Первый запуск

Если базы еще нет, скрипт сам создаст:
- папку `data/`
- файл `data/poly_panic.db`
- нужные таблицы SQLite

На самом первом запуске после создания базы:
- история по рынкам еще пустая;
- `top` обычно ничего не покажет;
- часть триггеров не сможет сработать, потому что не с чем сравнивать;
- новые рынки будут считаться относительно пустой базы.

Это нормальное поведение.

## Что видно в терминале

В начале прохода выводится:
- сколько рынков осталось после фильтров;
- сколько найдено новых;
- какие фильтры сейчас активны.

Если найдены аномалии, ниже будут блоки вида:
- тип алерта;
- вопрос рынка;
- текущий отслеживаемый исход и его вероятность;
- краткое описание причины;
- `slug` рынка.

## Логи

Технические логи пишутся в:

```text
logs/poly_panic.log
```

Там видно:
- старт приложения;
- активные фильтры;
- сколько рынков пришло из API;
- сколько рынков было отфильтровано и почему;
- сколько алертов сработало.
